## The one invariant that matters

> **Invariant (FIFO-first):** Every `Port` is realized as a FIFO path.
> Transports never define ports; they only *attach* to existing ports.
> Procedures never change the meaning of ports; they only converge reality to structure.

In Haskell: make it **impossible to represent** a non-FIFO port.

## 1) Collapse the runtime into 3 orthogonal axes

Your “logic cube” becomes three independent domains:

### Ports = Space (structural)

```haskell
newtype PortName = PortName Text
newtype FifoPath = FifoPath FilePath

-- FIFO-first: a Port IS a FIFO, not a sum type.
data Port = Port
  { portName :: PortName
  , fifoPath :: FifoPath
  } deriving (Eq, Show)
```

### Processes = Motion (dataflow over ports)

```haskell
newtype ProcName = ProcName Text
newtype Command  = Command Text

data Process = Process
  { procName :: ProcName
  , command  :: Command
  , waitsOn  :: [PortName]  -- reads from these FIFOs
  , firesTo  :: [PortName]  -- writes to these FIFOs
  } deriving (Eq, Show)
```

### Procedures = Time (control loop steps)

```haskell
data Procedure
  = Compile
  | Reconcile
  | Probe
  | Heal
  deriving (Eq, Ord, Show)
```

That’s your cube:

```
Port × Process × Procedure
```

Every event you emit sits at a coordinate in that cube.

## 2) Make “network sphere” a projection, not a primitive

## 3) Define the board as a pure, checkable structure

### Board well-formedness laws (the core)

This is where your “core invariant” becomes a *mathematical contract*.

```haskell
data BoardError
  = MissingPort PortName
  | DuplicatePortPath FifoPath
  | ProcRefsMissingPort ProcName PortName
  | TransportRefsMissingPort TransportName PortName
  deriving (Eq, Show)

validateBoard :: Board -> [BoardError]
validateBoard b =
  fifoUniqueness b ++ procPortsExist b ++ transportPortsExist b
  where
    fifoUniqueness :: Board -> [BoardError]
    fifoUniqueness Board{ports} =
      let paths = map fifoPath (Map.elems ports)
          dups  = duplicates paths
      in map DuplicatePortPath dups

    procPortsExist :: Board -> [BoardError]
    procPortsExist Board{ports,processes} =
      [ ProcRefsMissingPort pn p
      | (pn,proc) <- Map.toList processes
      , p <- waitsOn proc ++ firesTo proc
      , Map.notMember p ports
      ]

    transportPortsExist :: Board -> [BoardError]
    transportPortsExist Board{ports,transports} =
      [ TransportRefsMissingPort tn (attachPort t)
      | (tn,t) <- Map.toList transports
      , Map.notMember (attachPort t) ports
      ]
```

**This is the collapse.**
Everything else (reconcile, probe, heal) is implementation detail around preserving these invariants.

## 5) ULP polynomials as the algebraic core (free abelian group)

You already treat events as “polynomials over atoms”.
In Haskell, represent that as a `Map Atom Int` (free abelian group).

```haskell
newtype Atom = Atom Text
  deriving (Eq, Ord, Show)

newtype ULP = ULP (Map Atom Int)
  deriving (Eq, Show)

ulpAdd :: ULP -> ULP -> ULP
ulpAdd (ULP a) (ULP b) = ULP (Map.filter (/=0) (Map.unionWith (+) a b))

ulpNeg :: ULP -> ULP
ulpNeg (ULP a) = ULP (Map.map negate a)
```

This gives you the clean invariant:

> **ULP invariant:** Event meaning lives in an abelian group over atoms.
> Trace analysis is group algebra, not string parsing.
